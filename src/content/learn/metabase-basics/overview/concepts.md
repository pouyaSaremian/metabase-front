# مفاهیم متابیس

نمای کلی سریع از مفاهیم اصلی در متابیس.

## نمای کلی

متابیس یک پلتفرم "هوش کسب‌وکار" (BI) است که ابزارهای زیادی برای درک و اشتراک‌گذاری داده شما به شما می‌دهد. شرکت‌ها معمولاً از متابیس برای دادن راه آسان به تیم‌های خود برای پرس‌وجوی داده استفاده می‌کنند، یا برای جاسازی متابیس در اپلیکیشن خود تا به مشتریان اجازه کاوش داده به خودی خود را بدهند.

> از ابزار BI دیگری می‌آیید؟ [راهنماهای انتقال](../../cheat-sheets/transition-guides/index.html) ما را بررسی کنید.

می‌توانید از متابیس برای:

- پرس‌وجوی پایگاه داده خود با استفاده از:
  - یک [query builder گرافیکی](#query-builder)، یا
  - [ویرایشگر پرس‌وجوی بومی](#native-query-editor)
- مشاهده نتایج به عنوان جداول یا نمودارها (که می‌توانید سفارشی کنید)
- ذخیره نتایج به عنوان [سؤال‌ها](#questions) که می‌توانند به:
  - [داشبوردها](#dashboards)
  - [مجموعه‌ها](#collections)
اضافه شوند
- افزودن چندین سؤال به داشبوردها.
- افزودن فیلترهای داشبورد که همه سؤال‌ها را به یکباره به‌روزرسانی می‌کنند.
- سازماندهی سؤال‌ها و داشبوردها در مجموعه‌ها.
- ایجاد [مدل‌ها](#models) برای گردآوری مجموعه داده‌ها، آسان‌تر کردن پرس‌وجوی داده‌های معمولاً فیلتر شده یا join شده برای مردم.
- ایجاد [معیارها](#metrics) برای استاندارد کردن نحوه محاسبه اعداد مهم توسط تیم شما.

## مفاهیم اصلی

متابیس ابزارهای زیادی دارد، اما در اینجا جعبه ابزار پایه است:

- [**سؤال‌ها**](#questions) پرس‌وجوهای ذخیره شده با تجسم‌ها هستند که می‌توانید به داشبوردها یا مجموعه‌ها اضافه کنید. سؤال‌ها نمودارهایی هستند که می‌توانید در داشبوردها سازماندهی کنید.
- [**داشبوردها**](#dashboards) سؤال‌های مرتبط (نمودارها و کارت‌های دیگر) را گروه‌بندی می‌کنند که می‌توانند با هم فیلتر و تازه‌سازی شوند.
- [**مجموعه‌ها**](#collections) مثل پوشه‌ها برای سازماندهی و مدیریت مجوزها برای سؤال‌ها، داشبوردها و موارد دیگر شما هستند.
- [**مدل‌ها**](#models) مثل viewها هستند که داده را از پایگاه داده شما گردآوری می‌کنند.
- [**معیارها**](#metrics) روش رسمی برای محاسبه اعداد مهم برای تیم شما را تعریف می‌کنند.

## سؤال‌ها

![سؤال](../../images/metabase-concepts/question.png)

سؤال‌ها پرس‌وجوهای ذخیره شده به علاوه تجسم آن‌ها هستند (می‌توانید بین یک جدول و یک نمودار جابه‌جا شوید). اگر از [Tableau](../../cheat-sheets/transition-guides/tableau-to-metabase.html) می‌آیید، سؤال‌ها مثل worksheetها هستند؛ اگر [Power BI](../../cheat-sheets/transition-guides/powerbi-to-metabase.html)، آن‌ها مثل Reportها هستند.

همچنین می‌توانید:

- [تنظیم هشدارها](../../../docs/latest/questions/alerts.html) روی سؤال‌ها برای اطلاع‌رسانی وقتی داده شما شرایط خاصی را برآورده می‌کند.
- [صادر کردن نتایج](../../../docs/latest/questions/exporting-results.html) سؤال‌ها به CSV، XLSX، یا JSON (یا PNG برای نمودارها).

دو روش اصلی برای ایجاد سؤال‌ها وجود دارد: query builder، و ویرایشگر کد بومی:

### Query Builder

![Query builder](../../images/metabase-concepts/query-builder.png)

Query builder به شما اجازه ایجاد سؤال‌ها بدون دانستن SQL را می‌دهد. یک رابط point-and-click ارائه می‌دهد که در آن می‌توانید:

- داده‌ای که می‌خواهید تحلیل کنید را انتخاب کنید.
- [فیلتر](../../../docs/latest/questions/query-builder/filters.html) به مقادیر یا شرایط خاص.
- [خلاصه و گروه‌بندی داده](../../../docs/latest/questions/query-builder/summarizing-and-grouping.html)، مرتب‌سازی، و افزودن ستون‌های سفارشی.
- [Join](../../../docs/latest/questions/query-builder/join.html)، مرتب‌سازی، و افزودن ستون‌های سفارشی.
- [ایجاد تجسم‌ها](../../../docs/latest/questions/visualizations/visualizing-results.html) از نتایج خود.

حتی متخصصان SQL اغلب از query builder استفاده می‌کنند چون:

- سریع‌تر است. می‌توانید از نمودارها drill through کنید، نتایج را گروه‌بندی کنید، و روی یک سؤال تکرار کنید فقط با کلیک کردن.
- متابیس یک نمودار برای شما انتخاب می‌کند (که می‌توانید به صورت دستی تغییر و سفارشی کنید).
- نمودارهایی که query builder تولید می‌کند تعاملی هستند: می‌توانید [از نمودارها drill through کنید](../querying-and-dashboards/questions/drill-through.html) برای کاوش بیشتر (برخلاف نمودارهای ساخته شده با ویرایشگر کد بومی).
- می‌توانید سؤال را به افرادی که SQL نمی‌دانند تحویل دهید.
- به طور شگفت‌انگیزی قدرتمند است: [عبارات سفارشی](../../../docs/latest/questions/query-builder/expressions.html) را ببینید.

### ویرایشگر پرس‌وجوی بومی

![ویرایشگر پرس‌وجوی بومی](../../images/metabase-concepts/native-query-editor.png)

اگر SQL (یا زبان پرس‌وجوی پایگاه داده خود) را می‌دانید، همچنین می‌توانید سؤال‌ها را با ویرایشگر بومی ایجاد کنید. می‌توانید:

- پرس‌وجوهای پیچیده با کد قابل استفاده مجدد ذخیره شده به عنوان [snippetها](../../../docs/latest/questions/native-editor/snippets.html) بنویسید.
- از [پارامترها](../../../docs/latest/questions/native-editor/sql-parameters.html) برای پویا و قابل استفاده مجدد کردن پرس‌وجوهای خود استفاده کنید.
- به [مدل‌ها](../../../docs/latest/data-modeling/models.html) و [سؤال‌های ذخیره شده](../../../docs/latest/questions/native-editor/referencing-saved-questions-in-queries.html) در SQL خود ارجاع دهید.
- از توابع خاص پایگاه داده استفاده کنید.
- کارهایی انجام دهید که ممکن است در query builder ممکن نباشد.

**یک اشکال در مقایسه با query builder وجود دارد**: برخلاف سؤال‌های ساخته شده با query builder، مردم *نمی‌توانند* [از نمودارهای شما drill through کنند](../querying-and-dashboards/questions/drill-through.html).

## داشبوردها

![یک داشبورد در متابیس](../../images/metabase-concepts/dashboard.png)

داشبوردها راهی برای گروه‌بندی و ارائه سؤال‌های مرتبط هستند.

با داشبوردها، می‌توانید:

- [چیدمان چندین سؤال](../../../docs/latest/dashboards/introduction.html) در یک چیدمان که منطقی است.
- [افزودن فیلترها](../../../docs/latest/dashboards/filters.html) که چندین سؤال را به یکباره تحت تأثیر قرار می‌دهند.
- [افزودن کارت‌های متنی](../../../docs/latest/dashboards/introduction.html#adding-headings-or-descriptions-with-text-cards) برای ارائه زمینه و توضیحات.
- [تنظیم فواصل تازه‌سازی خودکار](../../../docs/latest/dashboards/introduction.html#auto-refresh).
- [کارت‌ها را تعاملی کنید](../../../docs/latest/dashboards/interactive.html).
- [تنظیم اشتراک‌ها](../../../docs/latest/dashboards/subscriptions.html) برای ارسال خودکار داشبوردها از طریق ایمیل، Slack، یا webhook.

## مجموعه‌ها

![مجموعه‌ها](../../images/metabase-concepts/collections.png)

مجموعه‌ها مثل پوشه‌ها کار می‌کنند. آن‌ها مثل سیستم فایل برای متابیس هستند. می‌توانید از مجموعه‌ها برای:

- گروه‌بندی محتوای مرتبط با هم: سؤال‌ها، داشبوردها، مدل‌ها، و معیارها. به عنوان مثال، برای گروه‌بندی همه موارد برای یک تیم خاص.
- علامت‌گذاری موارد به عنوان رسمی (ویژگی Pro).
- تودرتو کردن مجموعه‌ها در مجموعه‌های دیگر.
- افزودن [رویدادها و timelineها](../../../docs/latest/exploration-and-organization/events-and-timelines.html) برای ردیابی تاریخ‌ها و نقاط عطف کلیدی.

## مدل‌ها

[مدل‌ها](../../../docs/latest/data-modeling/models.html) مجموعه داده‌های تمیز و گردآوری شده شما هستند که به مردم کمک می‌کنند از پای درست شروع کنند. آن‌ها مثل یک صفحه‌گسترده خوب سازماندهی شده هستند که داده را از مکان‌های مختلف ترکیب می‌کند و محاسبات مفید اضافه می‌کند. می‌توانید مدل‌ها را برای:

- ترکیب و فیلتر داده از چندین جدول.
- شامل محاسبات مفید و فیلدهای تمیز شده.
- ارائه توضیحات واضح و انواع معنایی (مثل "قیمت").

ایجاد کنید.

به عنوان مثال، بگویید یک پایگاه داده e-commerce با جداول جداگانه برای سفارش‌ها، مردم، و محصولات دارید. می‌توانید یک مدل "سفارش‌های مشتری" ایجاد کنید که:

- ستون‌های معمولاً استفاده شده از این جداول مختلف را با هم join می‌کند.
- فیلدهای محاسبه شده مفید مثل "ارزش کل طول عمر" اضافه می‌کند.
- سفارش‌های تست را فیلتر می‌کند.
- توضیحات واضحی از معنای هر فیلد شامل می‌کند.

حالا هر کسی می‌تواند از مدل به عنوان نقطه شروع برای سؤال‌های جدید استفاده کند، بدون اینکه هر بار نیاز به دست و پنجه نرم کردن با داده زیربنایی داشته باشد.

## معیارها

![معیارها](../../images/metabase-concepts/metrics.png)

به جای اینکه همه اعداد مهم (مثل درآمد، کاربران فعال، و غیره) را به روش خود محاسبه کنند، می‌توانید این محاسبات را به عنوان معیارها استاندارد کنید.

با [معیارها](../../../docs/latest/data-modeling/metrics.html)، می‌توانید:

- یک منبع واحد حقیقت برای محاسبات مهم ایجاد کنید.
- این محاسبات را در هر سؤال، داشبورد، یا مجموعه شامل کنید.

به عنوان مثال، بگویید می‌خواهید درآمد شرکت خود را ردیابی کنید. می‌توانید یک معیار "درآمد ماهانه" ایجاد کنید که:

- داده را از جدول `orders` شما می‌گیرد.
- سفارش‌های لغو شده را مستثنی می‌کند.
- مجموع را جمع می‌زند.
- بر اساس ماه گروه‌بندی می‌کند.

به این ترتیب هر کسی می‌تواند از معیار "درآمد ماهانه" در سؤال‌ها و داشبوردهای خود استفاده کند، و شما با اعداد درآمد مختلف در داشبوردهای مختلف مواجه نخواهید شد.

## یک نکته آخر

`cmd + k` (Mac) یا `ctrl + k` (Windows/Linux) را فشار دهید تا command palette را باز کنید. می‌توانید از آن برای:

- جستجو در همه محتوای متابیس خود استفاده کنید.
- پرش به موارد خاص (سؤال‌ها، داشبوردها) یا صفحات (مثل تنظیمات Admin).
- ایجاد سؤال‌ها و داشبوردهای جدید.
- دسترسی به موارد اخیر.
- پیدا کردن مستندات.

سفر خوبی داشته باشید!
