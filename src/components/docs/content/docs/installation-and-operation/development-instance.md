# نمونه‌های توسعه (Development instances)

{% include plans-blockquote.html feature="Development instances" %}

> یک نمونهٔ توسعه **نباید در محیط تولید استفاده شود** و باید روی **متابیس نسخهٔ ۵۵ یا بالاتر** اجرا شود.

در پلن‌های Pro و Enterprise می‌توانید یک نمونهٔ توسعه یا توکن توسعه (development token) را به‌عنوان افزونه (add‑on) به پلن خود اضافه کنید.

- **Self‑hosted**: می‌توانید یک dev token تهیه کنید تا بابت هر حساب کاربری آزمایشی جداگانه صورتحساب دریافت نکنید. می‌توانید از همان dev token برای یک یا چند نمونهٔ توسعهٔ self‑hosted استفاده کنید (یعنی بدون هزینهٔ اضافی می‌توانید همان توکن را برای چندین dev instance به‌کار ببرید). این توکن توسعه به همان مجموعهٔ امکانات پولی‌ای دسترسی دارد که توکن تولید (production token) شما دارد.
- **Cloud**: می‌توانید یک نمونهٔ توسعه که روی [Metabase Cloud](https://www.metabase.com/cloud/) میزبانی می‌شود خریداری کنید. این نمونهٔ توسعه هم به همان مجموعهٔ قابلیت‌های پولی توکن تولید شما دسترسی خواهد داشت.

نمونه‌های توسعه فقط برای پلن‌های Pro و Enterprise در دسترس‌اند و بخشی از پلن‌های پایه نیستند. این نمونه‌ها باید پس از پایان دورهٔ آزمایشی و انجام اولین پرداخت، به‌عنوان add‑on به پلن شما افزوده شوند.

## پرداخت مبلغ ثابت به‌جای پرداخت به‌ازای هر کاربر تستی

این همان «پیشنهاد ارزش» اصلی است. برخلاف نمونه‌های تولیدی که در آن‌ها به‌ازای هر حساب کاربری پرداخت می‌کنید، برای یک dev instance تنها یک مبلغ ثابت می‌پردازید. به این ترتیب می‌توانید به‌راحتی با داشبوردها، سطح دسترسی‌ها و سایر تنظیمات بازی کنید، بدون این‌که مجبور باشید برای حساب‌هایی که فقط برای تست استفاده می‌کنید هزینه بپردازید.

وقتی از تغییرات در dev instance خود راضی بودید، می‌توانید آن‌ها را [صادر (export) و در نمونهٔ تولیدی خود وارد (import)](./serialization.md) کنید.

## صرفه‌جویی با توکن‌های dev حتی در حالت self‑hosted

درست است که هنگام self‑hosting می‌توانید از توکن تولید خود برای تست چند نمونهٔ متابیس استفاده کنید، اما در این حالت همچنان بابت حساب‌های کاربری تستی در هر کدام از این نمونه‌ها صورتحساب دریافت خواهید کرد. در چنین سناریویی، استفاده از توکن dev منطقی است، چون فقط یک مبلغ ثابت برای هر تعداد dev instance که می‌خواهید (و هر تعداد کاربر تستی که نیاز دارید) پرداخت می‌کنید.

## استفاده‌نکردن از نمونه‌های توسعه در محیط تولید

رابط کاربری نمونه‌های توسعه، از جمله رابط همهٔ خروجی‌ها (exports) و اعلان‌ها، دارای واترمارک است.

![Watermark banner](./images/watermark-banner.png)

## ایجاد یک نمونهٔ توسعه

اگر روی پلن Pro یا Enterprise هستید، می‌توانید در Metabase Store یک نمونهٔ توسعهٔ متابیس خریداری کنید:

1. وارد [حساب خود در Store](https://store.metabase.com) شوید.
2. به تب **Instances** بروید.
3. روی **+ New instance** کلیک کنید.
4. گزینهٔ **Development instance** را انتخاب کنید.

![Create development instance](./images/create-development-instance.png)

در این مرحله می‌توانید بین یک نمونهٔ Cloud یا دریافت توکن برای یک نمونهٔ توسعهٔ self‑hosted یکی را انتخاب کنید.

## داون‌گرید همراه با نمونه‌های توسعه

اگر روی پلن Pro Cloud هستید و به پلن Starter داون‌گرید کنید، همهٔ نمونه‌های توسعه‌ای که ساخته‌اید حذف می‌شوند. همهٔ نمونه‌های تولیدی شما به‌طور کامل باقی می‌مانند (البته بدون قابلیت‌های Pro).

اگر داون‌گرید کنید و روی پلن self‑hosted باشید، توکن توسعه دیگر معتبر نخواهد بود و دسترسی شما به قابلیت‌های پولی قطع می‌شود.

## انتقال داده‌ها از محیط توسعه به تولید

می‌توانید تغییراتی را که در نمونهٔ توسعهٔ خود اعمال کرده‌اید صادر کنید و در نمونهٔ تولیدی وارد کنید. برای جزئیات بیش‌تر، راهنمای [serialization](../installation-and-operation/serialization.md) را ببینید.
